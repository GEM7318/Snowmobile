Using auto_run ¶

Section Take-Away

The SQL object has an autorun attribute which determines whether the SQL generated by a given method is returned as a string or executed on the current connection.

The default value of the autorun is True , meaning it will execute the generated SQL as opposed to returning it as a string.

The behavior demonstrated below encompasses all methods of snowmobile.SQL , not just snowmobile.SQL.table_sample() .

Let's instantiate a Connector object and create a dummy table to use for the rest of the example.

Lines 11-12 in the below make use of snowmobile.Script to parse a sql file containing a single statement and and execute it to keep from cluttering the snippet with the full query; the contents of the table created are the only thing that matters, but the script is included in the dummy_table.sql tab for clarity.

Demonstrate core functionality of snowmobile.SQL object.
../docs/examples/mod_sql/sql_working_example.py

import snowmobile

from pathlib import Path

sn = snowmobile.Connect()

path = Path.cwd() / 'docs/examples/mod_sql/dummy_table.sql'
_ = snowmobile.Script(path=path, sn=sn).run(1)

1: When auto_run = True ¶

By default, calling the method on our dummy table will execute the generated query and return the results in a DataFrame . 

16
17
18
19
 
sample1 = sn.sql.table_sample('dummy_table', n=5)

print(type(sample1))   #> <class 'pandas.core.frame.DataFrame'>
print(sample1.shape)   #> (5, 3)

Whether set to True or False , the behavior imposed by the snowmobile.SQL.auto_run can be superseded by the run keyword argument provided to any method: 

21
22
23
 
sample1_run_false = sn.sql.table_sample('dummy_table', n=5, run=False)

print(type(sample1_run_false))  #> <class 'str'>

With the contents of sample_run_false: str being: 

24
 
print(sample1_run_false)

>>>
select
    *
from SANDBOX.SAMPLE_TABLE
limit 5

1: When auto_run = False ¶

To reverse this behavior, auto_run can be set to to False with: 

28
 
sn.sql.auto_run = False

With auto_run disabled, the run=False argument can be omitted to produce the same string of sql created on line 21 above: 

30
31
32
33
 
sample2 = sn.sql.table_sample('dummy_table', n=5)

print(type(sample2))                 #> <class 'str'>
print(sample2 == sample1_run_false)  #> True

To get the DataFrame created on line 16 , we now need to explicitly pass run=True to the method:

sample2_run_true = sn.sql.table_sample('dummy_table', n=5, run=True)

print(type(sample2_run_true))            #> <class 'pandas.core.frame.DataFrame'>
print(sample2_run_true.equals(sample1))  #> True
