All configuration options for establishing connections to Snowflake

The credentials (alias) to use by default if not specified in :arg:`snowmobile.Connector.creds`

Groups subsections of credentials, each declared with the structure of [connection.credentials.credentials_alias]

The value of credentials_alias is the literal string to pass to the creds argument of snowmobile.Connector to establish the Snowflake connection with those credentials.

Additional keyword-arguments can be specified in an aliased section so long as they are provided verbatim as they should be passed to the snowflake.connector.connect() method; this can be used to to map a specific timezone or transaction mode (for example) to a specific set of credentials.

All configurations relating to loading data

Default file format to use when loading local data into the warehouse

This is similar to the connection.default-creds except that instead of referring to a credentials alias, it's grouping together aliases/names of:

  1. An in-warehouse file format

  2. A set of [loading.export-options] (below)

  3. A statement tag associated with the DDL that creates this file format (to be fetched by snowmobile if it's called on and doesn't yet exist in the current schema)

Groups subsections of export-options

Default file-export options for snowmobile_default_csv

This defines a set of keyword arguments associated with snowmobile_default_psv to be passed directly to the pandas.DataFrame.to_csv() method when exporting data to a local file before loading into a staging table via Snowflake 's put file from stage() statement.
